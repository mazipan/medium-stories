Clash of UI Designer vs Frontend Developer

I am Frontend Developer so content in this post is subjective by frontend developer side, some content is just opinion and not really fact that happen in your condition.

UI Designer and Frontend Developer

UI designer is person that have responsibility to build User Interface of some software/website applications, they will make as good as UI for the end user. In some company this person will also have title UX Designer that mean they will responsible in not just User Interface but more wide in User Experience. Person that have two title like that will have more knowledge to deliver good software/website that easy to use, simple step, and showing right information to end user. UI Designer result represents with Mockup/Sketch of software/application that will be delivered.

Frontend Developer is person that translated all design that have built by UI Designer to the real software/application by writing lines of code in any programming language. Frontend Developer is also translated all interaction flow that built by UX Designer to the code.

Clash of them

Every job that meet two different field is always giving some clash because different perception and different knowledge of them. UI Designer and Frontend Developer coming from two different field, UI Designer usually coming from Product/Design and Frontend Developer usually coming from IT field. This two field have many differentiate that make it hard enough to always speak in one word.

Here some of clash that happen :

From UI Designer side :

  1. UI Designer giving design that too hard to implement in line of code.
  2. UI Designer not delivering detail of specs in their design.
  3. Too much creativity that make lose of consistency in whole product.
  4. Not considering any error state and empty state.
  5. Not really care about performance of that product.

From Frontend Developer side :

  1. Frontend Developer not delivering final result and all interaction flow as UI Designer has described.
  2. Frontend Developer is too lazy to inspect any detail pixel and other requirement from design.
  3. Frontend Developer is too much complain about the design without reasonable explanation.
  4. Have no good communication with UI Designer if there is any flow/state that not accommodate in design.
  5. Too much refer to the framework that used although is not fit with flow/design.
  6. Too much aware with performance that not proven will be affected.

Solution

Whatever clash that happens between this two field but should never make reason to not delivering a good product. In fact, this two field has same mission to make product much better by each own knowledge. UI Designer trying to delivering best experience to end user and Frontend Developer trying to delivering as same as UI Designer has described.

This two field must know that good product is start by good planning including design process, so Frontend Developer should have contribution in design phase to make clash in development phase is reduced.

But even with good planning it was never make all the clash is gone away. UI Designer and Frontend Developer should have ownership to the product, any clash that happens should can communicate between them. Should always there space for negotiation if clash happen and the solution should refer to end user happiness not by each ego.
