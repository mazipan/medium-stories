SASS + BEM for Better CSS Development

Intro

Yes I am Frontend developer, my daily job is about building a block of HTML then adding CSS code for more humanity and fill with JavaScript.

I have code frontend at least from 2013 and I was an orthodox frontend that never know any tools for supporting and helping finish my job easier. I just know HTML, CSS and JavaScript, That’s it.

But everything was changed after fire nation attacked.

Building websites nowadays is not enough if you’re only using HTML, CSS and JavaScript although the browser until today was only know those. And as a developer, we should facing the changing as a challenge. When we think our existing solution is not valid again then maybe we should looking for another solution. 

Why Using SASS ?

In case of CSS development changing, now almost of existing frontend developer have use pre-processor to help in CSS development. So do I, first I trying to compare existing pre-processor in the market that more fit with my style and needed. I found LESS, Stylus, and SASS as a good candidate at that time. LESS and Stylus could be the winner because they run in Nodejs engine, SASS basically run in Ruby Engine and It’s too much for frontend developer should have many engine for its development. But unfortunately SASS also have extended library called libsass that translated into many language, one of them is Nodejs. So, all of them now is side by side.

Let see into its syntax, Stylus has syntax that hard to learn, especially in tabulation sensitive usage. SASS have two different syntax called native SASS and SCSS syntax. SCSS syntax is same with native CSS syntax, very ease to learn for poor frontend developer like me. In syntax I think SASS and Less become winner for me because it’s simple learning curve.

SASS also more mature for pre-processor than its competitor with 9 year experience in market. 

So far I decide to choose SASS as my pre-processor, this is a subjective opinion you may have different result with different reason.

SASS Implementation

SASS like another pre-processor is just tools for helping Us in development phase, but for getting the good of pre-processor we still need to decide what best structure and standard that fit with your needed.

In first implementation I only seperate folder into mixin and fundamental, all file that represents to each pages is outside this two folder. I haven’t implemented any SASS syntax, I only use import for cut file into small piece and mixin to vendor prefix definitions.

I found that SASS is helping, but need more deep learning to feel more the SASS effect. Then I start to learn more about SASS syntax that will help me more in development.

BEM Convention

As I said above that SASS is great tool but it’s just a tool, I need more standard that make my CSS code easier to be managed, more modular, easier to be tracked. 

And BEM coming to solve all of those.

BEM unlike SASS that is tool for helping CSS development, BEM is just standard for class naming, So with BEM you will have standard in your CSS classname.

Why BEM standards helping me?

BEM is stand for Block, Element, Modifier. BEM change our habit in code CSS, if we use to code class name that reuse everywhere BEM won’t do that thing. BEM see all the thing as Block (*or you can see as component if you familiar with React), so if you want reuse anything you must reuse that Block not any little part of that.

BEM helping in manage my structure to be more modular, every BEM block now stand for single file SCSS that can reuse. BEM also make my code more trackable, because all class that show in HTML is always relevant with your filename. You can guess what file is it from class name in HTML with ease.

Conclusion

SASS + BEM is my favorite combination for me to developing CSS, it’s maybe not the best solution for you. But I hope you can get the big picture. You can choose any tools combination for helping you. There’s so much great combination that you can choose out there. Maybe you can look for Less + OOCSS ?
