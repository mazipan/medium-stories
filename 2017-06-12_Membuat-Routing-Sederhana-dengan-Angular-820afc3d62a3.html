Membuat Routing Sederhana dengan Angular

Kita akan membahas mengenai cara membuat Routing menggunakan framework Angular. Routing di frontend javascript framework memang seperti sudah menjadi feature yang diwajibkan ada jaman sekarang, Angular dalam hal ini seperti biasa yang tidak mengikuti cara framework modern lain dalam menghadirkan Routing nya yang memilih dipisah dari framework utamanya, Angular justru selalu hadir dengan paket lengkap termasuk dengan built-in Routing didalamnya. Pada versi 2.x sendiri Angular memisahkan repository untuk Routing sehingga Anda akan melihat depedency @angular/router pada package.json, namun bedanya bahwa Router pada Angular di maintain secara official oleh team mereka dan selalu release sebaga satu kesatuan framework bersamaan dengan core nya.

Bagi Anda yang baru saja bergabung, artikel ini merupakan serial dari belajar Angular. Anda bisa memulai dengan membaca bagian 1, bagian 2, dan bagian 3 terlebih dahulu sebelum meneruskan membaca artikel ini.

Kode diatas agak aneh karena saya menempelkan class active pada li bukan pada a, jadi saya butuh dua routerLink bertumpuk, ini bisa diubah tergantung kondisi Anda.

Sekian tulisan dari saya, masih banyak lagi fitur Routing dari Angular yang belum sempat kita ulik bareng-bareng seperti melempar parameter pada router, membuat nested router, membuat router tanpa hash, dll. Router pada Angular versi 2.x kali ini bisa dibilang cukup memuaskan dan bisa bersaing dengan berbagai fremework modern lainnya.
