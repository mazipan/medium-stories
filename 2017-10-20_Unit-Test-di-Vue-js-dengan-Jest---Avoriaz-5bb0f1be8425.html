Unit Test di Vue.js dengan Jest + Avoriaz

Belajar membuat unit testing sederhana untuk Vue.js component dengan Jest dan Avoriaz

Sepertinya pada tulisan kali ini saya tidak akan menjelaskan lagi kenapa kita perlu Unit Test di kode kita, kecuali kalian yakin 100% bahwa kode yang kalian bikin tidak akan menimbulkan bug dikemudian hari maka Unit Test menjadi wajib hukumnya di dalam project kita. Sebelum ini saya lebih banyak berkecimpung dengan kode Java yang jelas mudah untuk membuat Unit Test, tapi di Javascript khususnya Vue.js ini merupakan hal baru bagi saya. Jadi saya akan berbagi beberapa cerita mengenai beberapa implementasi Unit Test di project open source saya.

Sebelumnya pada saat saya pertama kali belajar Vue.js dan generate project menggunakan Vue-CLI dengan template webpack, saya belajar menggunakan Unit Test di dalam project Vue.js namun masih menggunakan Karma sebagai Runner nya kemudian menggunakan bantuan chai atau karma-sinon-chai untuk assert library nya serta instanbul sebagai coverage report.

Kekurangan dari karma runner ini adalah setup yang terlalu banyak, untuk mengakomodasi kekurangan ini kita coba melihat alternatif lain yakni Jest yang menawarkan konfigurasi yang lebih sederhana dan sudah termasuk dengan assert dan expect bahkan coverage di dalamnya, sehingga mengurangi terlalu banyak dependency yang harus di install dimana sering kali membuat malas untuk memperbarui versi bila ada update dari masing-masing dependency tersebut.

Secara fitur sederhana unit test di dalam Vue.js bawaan sebenarnya sudah cukup baik, namun ada beberapa kasus yang membuat unit test menggunakan cara seperti ini menjadi susah untuk di jalankan. Salah satunya adalah membuat mocking dan isolated unit testing, jadi beberapa kasus kita membutuhkan mock data alih-alih menggunakan komponen asli. Pun begitu dengan isolated unit testing dimana kita ingin membuat unit test yang independen di masing-masing komponen tanpa harus terikat dengan komponen lain. Untuk menangani beberapa kasus yang lebih susah ini kita coba menilik salah satu unit test library yang lumayan populer di kalangan Vue.js developer, yakni Avoriaz.

eddyerburgh/avoriaz
avoriaz - 🔬 a Vue.js testing utility librarygithub.com

Di dalam Avoriaz terdapat banyak sekali helper untuk membuat unit test sehingga proses membuat unit test ini akan terasa lebih mudah dan tidak terasa sebagai beban tambahan bagi kita.

Berikut beberapa langkah yang Kevin Ongko buat dalam setup unit test di project saya vue-select-image :

Install dependency

Beberapa dependency tambahan yang perlu kita install antara lain avoriaz, jest, jest-vue-preprocessor. Anda bisa menginstall dependency tersebut dengan npm ataupun yarn :

npm i avoriaz jest jest-vue-preprocessor — save-dev
yarn add avoriaz jest jest-vue-preprocessor — dev

Konfigurasi Jest

Ini salah satu yang menarik, karena biasanya kita harus setup banyak hal untuk bisa memasang sebuah unit test runner di Javascript. Dengan Jest hal ini dibuat sesederhana mungkin, berikut contoh konfigurasi Jest di repository vue-select-image yang diletakkan di file package.json karena memang sangat sederhana :

Kita mendefinisikan directory untuk meletakkan coverage report dengan coverageDirectory, hal ini bisa dilewatkan karena Jest sudah membuat default directory bila kita tidak mendefinisikan hal ini. moduleNameMapper merupakan konfigurasi seperti layaknya alias yang kita buat di webpack config kita. moduleFileExtensions merupakan ekstensi dari file-file yang akan di baca nanti pada unit test kita. transform ini seperti loader pada webpack, kita menggunakan babel-jest bawaan Jest untuk file .js dan menambahkan jest-vue-preprocessor untuk file .vue kita. Ada alternatif lain untuk file .vue ini seperti jest-vue yang notabene juga buatan eddyerburgh yang merupakan pembuat Avoriaz juga.

Membuat Unit Test

Kita bisa membuat unit test kita di dalam folder test ataupun __test__ seperti direkomendasikan Jest. Di dalamnya kita membuat file .js dengan nama file yang di tambahkan .spec.js ataupun .test.js pada bagian belakang nama file agar Jest bisa otomatis menganggap file tersebut sebagai file unit test kita nantinya.

Dengan Avoriaz kita bisa melakukan mounting dan melakukan full render sebuah Vue komponen tanpa harus meng-extend dari Vue Instance, kita juga bisa memilih menggunakan shallow dari Avoriaz yang akan membuat Vue komponen beserta semua child yang berkaitan dibawahnya. Kedua metode ini akan memberikan nilai balik sebuah wrapper yang bisa kita gunakan untuk melakukan unit test pada fungsi dan perubahan pada Vue komponen kita.

Menggunakan Jest juga kita mengeliminasi sinon, karena sudah ada fungsi untuk Spy dalam Jest seperti terlihat pada gambar diatas.

Menjalankan Unit Test dan Coverage

Untuk menjalankan unit test bisa dengan mudah dengan perintah

jest — coverage

Perintah ini bisa kita letakkan di dalam script di package.json agar lebih mudah dijalankan nantinya.

Otomatisasi dan upload coverage

Untuk otomatisasi menjalankan unit test setiap kali ada commit yang dilakukan ke dalam repository kita akan menggunakan layanan dari Travis, sedangkan untuk mengukur coverage kita akan menggunakan layanan Codecov.
