Mempercepat Load Time #WWWID RSS Reader Menggunakan Framework Vue.js #WWWIDChallenge

Membongkar apa yang saya lakukan di tantangan #WWWIDChallenge dalam rangka mempercepat load time WWWID RSS Reader dengan Vue.js dan Workbox

WWWID belum lama ini mencoba menantang para Web Developer untuk sama-sama belajar dan bereksplorasi dengan pengetahuan framework masing-masing dalam hal optimasi load page suatu halaman web. Bagi yang belum tau tantangan seperti apa yang dimaksud, maka silahkan mampir langsung ke sumurnya disini. Meskipun tantangan ini tidak akan mendapatkan hadiah namun menurut saya baik bagi kalian untuk ikut belajar, selain karena ada banyak hal yang mungkin belum kalian tau, ada banyak hal juga yang bisa di eksplorasi untuk optimasi sebuah load page halaman web.

Teknologi yang digunakan¬†:

  1. Vue.js
  2. Vue-Router

Kenapa saya memilih Vue¬†?, karena keseharian saya memang harus bergelut dengan framework hijau ini. Jadi apa salahnya bila mencoba mengaplikasikan apa yang sudah dipelajari selama ini sembari belajar lagi cara optimasi yang benar diatas framework Vue.js.

Saya juga menggunakan Vue-Router untuk kebutuhan pindah-pindah halaman tanpa refresh, saya mengeliminasi Vuex dari project ini karena tidak ingin menambahkan beban size pada bundle akhir nantinya.

Untuk PWA saya akan memanfaatkan Workbox dari Google dan memilih versi webpack plugin yakni workbox-webpack-plugin agar mudah diintegrasikan dengan build tools nantinya.

Dan Webpack menjadi bundle tools favorit yang akan membantu banyak hal dibagian optimasi file nantinya.

Bila ingin belajar beberapa best practice dalam optimasi webpack bisa baca disini. Berikut adalah beberapa cara yang saya gunakan dalam memanfaatkan webpack untuk optimasi¬†:

>> Define Environment

Dengan mendefinisikan environment yang sedang digunakan kita bisa menghilangkan kode yang tidak diperlukan.

>> Extract CSS dari Vue.js bundle

Kita memanfaatkan extract-text-webpack-plugin untuk mengeluarkan CSS kedalam file tersendiri, hal ini penting karena CSS di Javascript akan dianggap sebagai String yang susah untuk di minify. Karena didalam file¬†.vue kita bisa menuliskan CSS/Pre-processor di dalamnya maka untuk extract kita butuh meletakkan loader untuk extract CSS tepat dibawah vue-loader¬†:

>> Minify HTML Output

Kecilkan ukuran HTML yang akan dimuat dihalaman, pastikan semua inline kode CSS dan Javascript juga ikut dikecilkan ukurannya.

>> Extract Vendor script

Kita akan meng-extract semua kode yang berasal dari folder node_modules dengan menggunakan CommonsChunkPlugin. Hal ini perlu dilakukan karena script vendor ini biasanya akan dimuat di semua halaman, dan jika kita bisa memisahkan script itu kedalam file tersendiri maka lebih mudah bagi browser untuk meng-cache ketika kita mngunjungi halaman lain.

>> Inline webpack runtime script

Webpack memiliki runtime kode yang digunakan untuk menentukan memuat chunk-chunk yang bertebaran. Kita akan meng-extract kode tersebut dan karena file yang dihasilkan terlalu kecil maka saya memutuskan untuk meng-inject kedalam inline kode HTML.

>> Minify JS File

Memampatkan semua script Javascript yang telah kita buat sehingga kita bisa mendapatkan kode dengan ukuran yang lebih kecil untuk dimuat.

>> Optimise CSS output

Kode untuk memastikan webpack mengoptimalkan CSS yang kita hasilkan.

>> Gzip Compression

üêå Lazy¬†Routing

Seperti disebutkan diatas kita menggunakan Vue-Router untuk berpindah halaman dan akan membuat beberapa URL halaman antara lain¬†:

  1. Halaman Utama¬†: halaman pertaman yang akan dikunjungi dan berisi daftar artikel terbaru.
  2. Halaman Kategori¬†: halaman daftar artikel dalam satu kategori.
  3. Halaman Detail¬†: halaman detail artikel dari daftar yang di klik.
  4. Halaman Tentang¬†: halaman yang berisi informasi mengenai aplikasi ini.

Hal jelek dari membuat beberapa halaman adalah bundle file size yang akan membengkak, karenanya salah satu solusinya adalah membuat Lazy Route, artikel lebih lengkapnya bisa dibaca disini.

Karena saya tidak senang dengan chunk webpack yang biasanya menggunakan nama 0, 1, 2,¬†‚Ä¶ maka saya juga mendefinisikan nama chunk yang akan dibuat oleh webpack. Ini memudahkan kita mengenali chunk tersebut di production.

Dengan menerapkan lazy routing artinya ketika halaman itu dikunjungi maka hanya akan mengambil script yang dibutuhkan dihalaman tersebut dan tidak perlu mengambil halaman lain yang tidak atau belum dibutuhkan. Lazy routing ini baik untuk mengurangi load time karena juga memecah bundle ke per masing-masing route.

Hal yang masih belum bisa adalah lazy juga untuk CSS, itu kenapa pada project kali ini saya urung menggunakan scoped CSS yang biasanya selalu menjadi hal wajib bagi saya ketika membuat aplikasi diatas Vue.js. Setelah saya melihat kedalam main.hash.css masih terdapat CSS yang dimiliki oleh halaman lain. Lagipula setelah dipertimbangkan lagi, dengan menghindari scope CSS saya bisa menghemat beberapa text karena tidak perlu mendapatkan tambahan random attribute sebagai penanda scoping.

üò¥ Lazy¬†Image

Karena di halaman utama kita dituntut untuk menambahkan thumbnail dari masing-masing artikel yang kita dapatkan dari RSS, maka penggunaan Lazy Load untuk memuat gambar menjadi kebutuhan wajib.

Pada project ini, saya memutuskan menggunakan Vue.js Plugin vue-lazyload, dimana plugin ini bisa dengan mudah setup lazy load sebuah gambar bahkan dengan konfigurasi global yang akan terimplementasi ke keseluruhan aplikasi.
