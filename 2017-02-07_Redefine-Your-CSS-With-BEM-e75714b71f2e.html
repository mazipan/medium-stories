Redefine Your CSS With BEM

Do you ever feel confused when code CSS in a team ? Do you ever ask a general questions like, Who’s make this class name ? Who’s change my class that I defined ? Why you override my class with your !important thing ? What is for this class name created ? Who’s using this class name ? And many more confusing questions, but it’s fact. Almost all CSS developer that working in team has those questions.

CSS have many ambiguous, CSS have no clear definition by its default about how to write it. We have too many freedom in CSS about file name, classname, selector usage and almost everything. I have seen file CSS that named with style.css and inside it there is so many line of code that never known who’s using all of that code.

I used to code Java Programming too, so I used to code everything with clear standard, Java has good rule even in filename. I think we as CSS developer should upgrade our habit to be more strict and follow some Rules/Standards/Conventions to achieve better code collaboration.

And for all of this problem, I try to implement one of proven convention that available in market. BEM (http://getbem.com).

BEM Convention

BEM stands for Block-Element-Modifier. BEM is convention for class naming in your CSS code. It’s very simple definition because BEM (*like it’s name) is just separate it’s rule into three, Block, Element and Modifier. BEM will help you to organize your CSS to answer all ambiguous that asking by our questions first above. If BEM is just class naming convention, how BEM will help us ? to answer this question let’s understand how BEM works in your code.

Let’s start with understand one by one BEM’s rule.

— Block

Block is every block parent that can stand independent, have no dependent with any other block. Block has no rule for its writing, just like ordinary class name. But lets add more rule so we have no ambiguous in the future. Class name for Block should be simple, meaningful and represent with its function or its place in a page. Example : header, navigation, tabs, menu, footer, etc.

.header{
}
.navigation{
}
.tabs{
}
.menu{
}
.footer{
}

— Element

Element is a child of its block, every part inside block called element. Element has dependent with its parent and cannot stand by itself. Element always inside block parent and has rule to include its parent class name as prefix of class name with adding two underscore (__) as a separation from its block parent.

So if you have block parent .header, you may have write your element like this example :

.header{
}
.header__brand{
}
.header__title{
}

— Modifier

Modifier is a state that will change behavior or appearance from Block or Element. BEM has rule for modifier with add its element/block as prefix and two dash ( — ) as a separation. Example :

.button{
}
.button--large{
}
.button--disabled{
}

BEM Implementation

After we understand all of simple rule in BEM now we will try to learn how to organize BEM to make our CSS better.

BEM’s deadly feature is a vision that a block in BEM should be independent and all the things inside this block should be handled by each block without affecting other block. Understand this vision is easy but to implementing is not easy as that.

First thing, we should separate file for each block so every block can reduce its change effect to others. To be more discoverable we also should give file name with same like it’s block name. So when we see class .header in html we can fast tracking that class because always placed in header file also with other class in html.

Second thing, we must change our mindset that very usual to reuse class, because in BEM we not reuse class but block and rules. Its looks like the hardest part for every CSS developer. Its very strange for us, we used to using class in any place and any div we want. But lets follow it first and we will decide in the last are we fit and enjoy with this convention or this is will add big effort to us.

Third thing, maybe you will need pre-processor helper to implement BEM because will be many confusing thing if you decide using BEM without any pre-processor (*while it’s not impossible).

Last thing from me when you want implement BEM to your project is make sure all of your team member has same vision and knowledge about this convention. Because BEM is just convention, it’s not giving an error even we not follow its guidance. A commitment from team member to implement better CSS code collaboration is matter. Transfer knowledge for each member to have same understand in BEM way is also matter.

For further reading about BEM you can found in here.

Hope this post will help you understand how BEM works to achieve better CSS code.
