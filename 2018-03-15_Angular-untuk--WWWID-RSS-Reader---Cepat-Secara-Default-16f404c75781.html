Angular untuk #WWWID RSS Reader : Cepat Secara Default

Membedah proses pembuatan #WWWID RSS Reader menggunakan framework Angular

Angular merupakan framework Javascript yang dibekingi oleh Google dan dikembangkan oleh banyak orang dari Google serta dengan bantuan berbagai kontributor dari berbagai belahan dunia. Angular sendiri merupakan pemain lama dalam dunia per-framework-an Javascript yang dalam tahun-tahun belakangan telah bertransformasi mengikuti kebutuhan akan modern web development.

Angular: Cepat Secara Default

Banyak yang sangsi dengan framework Angular belakangan ini, ditambah dengan makin hype-nya framework biru dan hijau disebelah yang makin menambah keraguan banyak teman-teman untuk mengadopsi Angular di project mereka. Faktor sejarah juga berperan dalam hal ini, dimana AngulaJS (versi Angular sebelum di full rewrite) memang dikenal sebagai salah satu yang tidak begitu bersahabat baik dengan performance dan speed.

Namun saya bisa bilang kalau Angular sudah sangat layak untuk teman-teman coba bila performance dan speed merupakan salah satu faktor yang ingin kalian capai. Bahkan kita bisa menggunakan Angular tanpa banyak setup dan memang dari sananya sudah cukup mumpuni (*menurut saya). Tapi tentu pengetahuan teman-teman akan cara memuat suatu halaman website juga bisa membantu dalam hal optimasi baik dari sisi koding maupun konfigurasi.

Mengurangi Redundancy Code

Salah satu yang sering dilupakan teman-teman apalagi kalau sudah ngoding di level framework adalah lupa untuk kembali ke dasar. Seringkali kita memanfaatkan berbagai fitur yang tersedia di framework tanpa tau apa yang sebenarnya terjadi atau apa yang sebenarnya dihasilkan dari hal yang kita lakukan. Salah satu hal yang coba saya kurangi adalah reduncancy code atau kode ganda yang sebenarnya cukup menggunakan satu kode.

Angular mendukung CSS Module secara default, namun tentu hal ini bisa membuat kita lupa bahwa kita bisa berulang-ulang membuat hal yang sama cuma demi meraih scoping CSS. Berdasarkan pengalaman saya, CSS Module tidak akan begitu berguna bagi project yang menginginkan efisiensi serta hanya dibangun oleh satu orang. Karena itu memindahan semua CSS yang bersifat modular ke dalam Global CSS menjadi penting.

Cukup menambahkan nama file yang akan dijadikan Global CSS pada .angular-cli.json dan Angular-CLI akan secara otomatis memproses hal ini sebagai Global CSS namun tetap melalui proses optimasi seperti yang seharusnya seperti minify dll.

Angular juga memiliki fitur Service yang bisa digunakan bilamana ada kode yang seringkali dipanggil di beberapa tempat karena sifatnya yang Injectable. Penggunaan service secara baik dan ditempat yang benar tentu akan membantu kita mengurangi duplikasi kode yang seringkali secara sengaja atau tidak sengaja kita buat.

Menghindari Bloated Code

Menggunakan Angular yang secara default membawa Typescript bisa menimbulkan bloated code bila tidak tepat tepat sasaran terutama dalam mendeskripsikan berbagai data yang digunakan. Untuk itu data harus dideskripsikan secara tepat guna dan harus diminimalisir pemindahan data dari satu tipe ke tipe lain.

Workbox untuk PWA

Angular secara official telah membuatkan Module tersendiri untuk membuat PWA, yakni dengan menggunakan @angular/service-worker. Sayapun berkeinginan menggunakan module ini untuk PWA support, namun setelah berbagai percobaan dan eksperimen sepertinya untuk project ini saya akan kembali menggunaan Workbox sebagai tools yang membantu mendukung PWA.

Masih Banyak Potensi

Project ini sudah menambahkan beberapa cara optimasi untuk hacking baik first paint maupun first interactive time seperti memindahkan beberapa critical script maupun style menjadi inline, menambahkan dukungan lazy load pada gambar yang dimuat, compress gambar pada saat runtime request, caching data yang diminta, dan beberapa hal lainnya. Namun sangat tidak mungkin project ini bisa memenuhi hasrat saya dan kalian para penggemar optimasi. Maka saya dengan senang hati menerima bantuan kode, saran maupun kritik dari teman-teman karena saya pun menyadari apa yang saya kerjakan masih banyak kekurangan dan saya sangat yakin masih banyak potensi yang bisa dikerjakan teman-teman kedepannya.
